# 📘 راهنمای نصب و راه‌اندازی LeoBot

## معرفی LeoBot

LeoBot یک ربات مدیریت گروه حرفه‌ای برای روبیکا است که به‌صورت کاملاً
اختصاصی و مستقل نوشته شده. تمام دستورات آن فارسی و Reply-محور هستند
(بدون دستورات اسلش مانند `/ban`). امکانات اصلی:

- قفل‌های محتوایی (لینک، عکس، فیلم، فایل، ویس، موزیک، گیف، استیکر، فوروارد، لوکیشن، مخاطب، نظرسنجی، پست، استوری، تبلیغات، کلمات ممنوع)
- سیستم اخطار با اقدام خودکار (بن بعد از رسیدن به سقف اخطار)
- آنتی‌اسپم / آنتی‌فلود با سکوت خودکار موقت
- خوشامدگویی، خداحافظی، قوانین گروه
- فیلتر و پاسخ خودکار
- آمار گروه و لیست ادمین‌ها
- پنل مدیریت خصوصی در PV با رمز عبور (فقط برای مدیران مجاز)
- Broadcast به تمام کاربرانی که با ربات Start کرده‌اند
- بکاپ و بازیابی دیتابیس

### ⚠️ نکته‌ی فنی مهم درباره‌ی اتصال با Bot Token

LeoBot با **Bot Token رسمی روبیکا** اجرا می‌شود (نه لاگین با شماره تلفن).
اما API رسمی بات روبیکا هیچ متدی برای «حذف واقعی عضو از گروه» یا
«محدود کردن یک عضو خاص» ندارد — این محدودیت از طرف خود پلتفرم روبیکاست،
نه کتابخانه‌ی `rubpy`. به همین دلیل:

- دستور **«حذف»** (حذف پیام) کاملاً واقعی و بدون محدودیت کار می‌کند،
  چون متد رسمی `deleteMessage` در Bot API وجود دارد.
- دستورات **«بن»**، **«آنبن»** و **«سکوت»** به‌صورت نرم‌افزاری
  (Soft-Moderation) در دیتابیس خود LeoBot ثبت می‌شوند: بعد از بن/سکوت
  شدن یک کاربر، هر پیام بعدی او در گروه به‌طور خودکار با همان متد
  رسمی `deleteMessage` پاک می‌شود — یعنی عملاً کاربر دیگر نمی‌تواند در
  گروه پیام بفرستد، حتی بدون حذف واقعی عضویتش.
- برای اخراج کامل یک عضو از گروه، همچنان باید یک ادمین انسانی این کار
  را از داخل خود اپلیکیشن روبیکا انجام دهد؛ هیچ کتابخانه‌ای نمی‌تواند
  این محدودیت پلتفرم را دور بزند تا زمانی که خود روبیکا چنین متدی را
  به Bot API رسمی اضافه کند.

جزئیات کامل این تحلیل در کامنت‌های ابتدای فایل `core/client.py` نیز
آمده است.

## پیش‌نیازها

### ۱) نصب Python

LeoBot به **Python نسخه 3.10 یا بالاتر** نیاز دارد.

- ویندوز: از [python.org](https://www.python.org/downloads/) نسخه‌ی مناسب را دانلود و نصب کنید. حتماً گزینه‌ی
  `Add Python to PATH` را در زمان نصب فعال کنید.
- لینوکس (اوبونتو/دبیان):
  ```bash
  sudo apt update
  sudo apt install python3 python3-pip -y
  ```

برای بررسی نصب بودن Python:
```bash
python3 --version
```

### ۲) دانلود پروژه

پوشه‌ی `LeoBot` را در مسیر دلخواه خود قرار دهید (یا با `git clone` دریافت کنید
اگر پروژه را در یک ریپازیتوری گیت قرار داده‌اید).

### ۳) ساخت محیط مجازی (توصیه‌شده)

```bash
cd LeoBot
python3 -m venv venv
source venv/bin/activate      # لینوکس / مک
venv\Scripts\activate         # ویندوز
```

### ۴) نصب کتابخانه‌ها

```bash
pip install -r requirements.txt
```

این دستور کتابخانه‌های زیر را نصب می‌کند:
- `rubpy` — کتابخانه‌ی اتصال به روبیکا
- `python-dotenv` — خواندن فایل تنظیمات `.env`

اگر نصب `rubpy` با خطا مواجه شد، دستور زیر را جداگانه امتحان کنید:
```bash
pip install -U rubpy
```

## تنظیم فایل `.env`

فایل نمونه را کپی کنید:
```bash
cp .env.example .env      # لینوکس/مک
copy .env.example .env    # ویندوز
```

سپس فایل `.env` را با یک ویرایشگر متن باز کرده و مقادیر زیر را پر کنید:

### تنظیم Token بات

`BOT_TOKEN` را از پنل مدیریت بات‌های روبیکا دریافت کنید (داخل خود
اپلیکیشن روبیکا، بخش «بات‌ها»/«روبیکاپ»، گزینه‌ی ساخت بات جدید — یک
توکن رشته‌ای به شما داده می‌شود). این توکن را داخل `.env` قرار دهید:
```
BOT_TOKEN=123456:abcDEF...
```
**هیچ لاگین با شماره تلفن یا کد تایید پیامکی لازم نیست.**

### تنظیم Owner ID

`OWNER_ID` شناسه‌ی حساب روبیکای شما (مالک اصلی ربات) است. ساده‌ترین راه
برای پیدا کردن آن:
1. ربات را یک‌بار طبق مراحل بعدی اجرا کنید.
2. یک پیام هرچند ساده در PV یا یک گروه بفرستید.
3. شناسه‌ی شما در فایل `logs/leobot.log` یا در جدول `members`/`bot_users`
   دیتابیس ثبت می‌شود؛ آن را در `OWNER_ID` قرار دهید و ربات را دوباره اجرا کنید.
4. در صورت نیاز، شناسه‌ی سایر مدیرانی که باید به «پنل ادمین» دسترسی
   داشته باشند را هم با کاما در `PANEL_ADMIN_IDS` اضافه کنید.

### تنظیم رمز پنل ادمین

طبق تنظیمات پیش‌فرض پروژه، رمز پنل ادمین دقیقاً باید برابر باشد با:
```
GG777mahi
```
این مقدار در `.env.example` از پیش داخل `ADMIN_PASSWORD` قرار دارد. اگر
خواستید رمز دیگری استفاده کنید، مقدار `ADMIN_PASSWORD` را عوض کنید یا
(برای امنیت بیشتر) هش SHA-256 آن را در `ADMIN_PASSWORD_HASH` بگذارید و
خط `ADMIN_PASSWORD` را پاک کنید. برای تولید هش دستی:
```python
import hashlib
print(hashlib.sha256("رمز-من".encode("utf-8")).hexdigest())
```
رمز هرگز به‌صورت متن ساده داخل کد پایتون نوشته نشده و همیشه از `.env`
خوانده می‌شود (ر.ک. `config.py` و `core/security.py`).

## اجرای ربات

```bash
python main.py
```

چون اتصال با Bot Token است، هیچ مرحله‌ی تعاملی (شماره تلفن/کد تایید)
وجود ندارد؛ ربات بلافاصله شروع به دریافت پیام‌ها (Polling) می‌کند.

اگر همه‌چیز درست باشد، پیام‌های زیر را در ترمینال می‌بینید:
```
[INFO] [main] در حال راه‌اندازی LeoBot نسخه 2.0.0 ...
[INFO] [database] دیتابیس با موفقیت مقداردهی اولیه شد.
[INFO] [client] هندلر دریافت پیام با موفقیت روی BotClient ثبت شد.
[INFO] [events] Event handler های LeoBot با موفقیت ثبت شدند.
[INFO] [main] LeoBot با موفقیت روشن شد و در حال گوش‌دادن به پیام‌هاست.
```

اگر اتصال به هر دلیلی قطع شود، LeoBot به‌طور خودکار و با فاصله‌ی زمانی
افزایشی (شروع از `RECONNECT_DELAY_SECONDS`، حداکثر تا
`RECONNECT_MAX_DELAY_SECONDS`) دوباره تلاش می‌کند؛ نیازی به اجرای دستی
مجدد نیست.

## استفاده در گروه

ربات را به گروه خود اضافه و ادمین کنید (برای اینکه دستور «حذف» و
حذف خودکار پیام‌های کاربران بن/سکوت‌شده کار کند، باید در خود روبیکا
دسترسی حذف پیام داشته باشد).

نمونه دستورات:
```
قفل لینک
بازکردن لینک
لیست قفل ها
تنظیمات
آمار
قوانین
```

دستورات Reply-محور (روی پیام کاربر Reply بزنید و بنویسید):
```
بن
آنبن
رفع بن
اخطار حرف زشت نزن
سکوت
رفع سکوت
حذف
ادمین کن
ادمین کامل کن
حذف ادمین
```

`آنبن` و `رفع بن` هر دو یک دستور هستند (مترادف). همان‌طور که در بخش
«نکته‌ی فنی مهم» بالا توضیح داده شد، «بن» به‌صورت نرم‌افزاری عمل
می‌کند: پیام‌های بعدی کاربر حذف می‌شوند اما اخراج کامل او از گروه
نیازمند اقدام دستی یک ادمین انسانی در اپ روبیکاست.

## پنل ادمین در PV

1. به PV ربات بروید و «شروع» یا Start بزنید.
   - اگر شناسه‌ی شما در `OWNER_ID` یا `PANEL_ADMIN_IDS` باشد، دکمه‌ی
     «🔐 پنل ادمین» نمایش داده می‌شود.
   - کاربران عادی فقط پیام خوش‌آمد ساده می‌بینند و اصلاً این دکمه را
     نمی‌بینند (طبق نیاز امنیتی پروژه، دسترسی حساس فقط برای مدیران است).
2. روی دکمه‌ی «🔐 پنل ادمین» بزنید (یا بنویسید: `پنل ادمین`).
3. رمز عبور را ارسال کنید (پیش‌فرض: `GG777mahi`، مگر اینکه در `.env` عوض کرده باشید).
4. پس از ورود موفق، پنل دکمه‌ای زیر نمایش داده می‌شود:
   - 📢 اطلاع‌رسانی همگانی (ارسال پیام به همه‌ی کاربرانی که Start کرده‌اند)
   - 👥 تعداد کاربران / 📋 لیست کاربران
   - 📈 آمار استفاده (تعداد گروه‌ها، کاربران PV، مجموع پیام‌ها)
   - 🏘 گروه‌های بات (لیست گروه‌هایی که بات داخل آن‌هاست)
   - 📝 لاگ‌ها / 💾 بکاپ
   - ⚙ تنظیمات بات (فعال/غیرفعال‌سازی پیش‌فرض آنتی‌اسپم و آنتی‌فلود
     برای گروه‌های جدید، و تغییر متن پیام خوش‌آمدگویی PV)
   - 🔒 خروج

اگر رمز اشتباه وارد شود، بعد از چند تلاش ناموفق (`PANEL_MAX_LOGIN_ATTEMPTS`
در `.env`) دسترسی به‌مدت `PANEL_LOCKOUT_SECONDS` ثانیه قفل می‌شود.

## اجرای دائمی روی سرور

### روش ۱: با `screen` یا `tmux`
```bash
screen -S leobot
python main.py
# سپس Ctrl+A و D برای خروج بدون قطع کردن ربات
```

### روش ۲: با systemd (لینوکس، توصیه‌شده برای سرور)

فایل `/etc/systemd/system/leobot.service` بسازید:
```ini
[Unit]
Description=LeoBot Rubika Manager
After=network.target

[Service]
WorkingDirectory=/path/to/LeoBot
ExecStart=/path/to/LeoBot/venv/bin/python main.py
Restart=always
RestartSec=5
User=your-username

[Install]
WantedBy=multi-user.target
```

سپس:
```bash
sudo systemctl daemon-reload
sudo systemctl enable leobot
sudo systemctl start leobot
sudo systemctl status leobot
```

## رفع خطاهای رایج

| خطا | راه‌حل |
|---|---|
| `ModuleNotFoundError: No module named 'rubpy'` | `pip install -U rubpy` را اجرا کنید |
| `ModuleNotFoundError: No module named 'dotenv'` | `pip install python-dotenv` |
| `[CONFIG] مقدار الزامی 'BOT_TOKEN' ...` | مقدار `BOT_TOKEN` را در `.env` پر کنید (از پنل بات‌ساز روبیکا دریافت می‌شود) |
| `[CONFIG] مقدار الزامی 'OWNER_ID' ...` | مقدار `OWNER_ID` را در `.env` پر کنید |
| ربات در گروه پاسخ نمی‌دهد | مطمئن شوید بات به گروه اضافه شده، دسترسی لازم (ارسال/حذف پیام) دارد و دستورات را دقیقاً مطابق نمونه‌ها می‌نویسید |
| «بن» عضو را واقعاً از گروه اخراج نمی‌کند | طبیعی است — طبق توضیح بخش «نکته‌ی فنی مهم»، Bot API رسمی روبیکا متد حذف عضو ندارد؛ LeoBot فقط پیام‌های او را خودکار حذف می‌کند. اخراج واقعی باید توسط ادمین انسانی در اپ روبیکا انجام شود |
| دکمه‌ی «پنل ادمین» دیده نمی‌شود | شناسه‌ی PV شما باید در `OWNER_ID` یا `PANEL_ADMIN_IDS` باشد |
| ثبت هندلر روی BotClient ناموفق است / پیام خطای `on_update` | نسخه‌ی نصب‌شده‌ی `rubpy` را با `pip install -U rubpy` به‌روزرسانی کنید؛ در صورت تغییر امضای متد، فایل `core/client.py` (متد `register_update_handler`) را مطابق نسخه‌ی جدید اصلاح کنید |
| اتصال مدام قطع/وصل می‌شود | مقدار `RECONNECT_DELAY_SECONDS`/`RECONNECT_MAX_DELAY_SECONDS` را در `.env` بررسی کنید؛ ربات به‌طور خودکار و پیوسته تلاش مجدد می‌کند و لازم نیست دستی ری‌استارت شود |
| دیتابیس قفل شده (`database is locked`) | ربات را کاملاً ببندید و دوباره اجرا کنید؛ از باز کردن همزمان دو نمونه از ربات خودداری کنید |

## توضیح ساختار فایل‌ها

```
LeoBot/
├── main.py                 نقطه ورود؛ راه‌اندازی دیتابیس، کلاینت و handler ها
├── config.py                خواندن و اعتبارسنجی تنظیمات از .env
├── requirements.txt          لیست کتابخانه‌های مورد نیاز
├── .env.example               نمونه‌ی فایل تنظیمات
├── INSTALL.md                 همین راهنما
│
├── core/
│   ├── client.py             Wrapper روی rubpy.bot.BotClient (Bot Token)؛ تنها نقطه‌ی تماس خام با API + Bridge سینک/اسینک
│   ├── events.py              مسیریاب اصلی رویدادها/پیام‌ها
│   ├── permissions.py          سیستم سطح دسترسی (Owner/Creator/Full Admin/Admin/User)
│   ├── security.py             احراز هویت پنل، تشخیص فلود/تکرار
│   └── logger.py               سیستم لاگ مرکزی (فایل + دیتابیس)
│
├── database/
│   └── database.py             تمام عملیات SQLite (CRUD)، شامل جدول‌های bot_settings و banned_users
│
├── modules/
│   ├── admin.py                بن/آنبن/سکوت/رفع‌سکوت/حذف (بن و سکوت به‌صورت Soft در دیتابیس)
│   ├── locks.py                 قفل‌های محتوایی
│   ├── anti_spam.py             آنتی‌اسپم/آنتی‌فلود/اجرای سکوت
│   ├── filters.py                مدیریت فیلترهای سفارشی
│   ├── warnings.py               سیستم اخطار
│   ├── welcome.py                خوشامدگویی و قوانین
│   ├── goodbye.py                خداحافظی
│   ├── auto_reply.py             اجرای پاسخ خودکار بر اساس فیلترها
│   ├── settings.py               نمایش/تغییر تنظیمات گروه
│   ├── statistics.py             آمار گروه
│   ├── owner_panel.py            پنل خصوصی ادمین‌ها در PV (چندمدیره، رمز از env)
│   └── broadcast.py              ارسال پیام همگانی
│
└── utils/
    └── helpers.py                توابع کمکی مشترک (پارس متن، استخراج Reply و ...)
```

## توسعه‌ی بیشتر

- برای افزودن دستور جدید، یک تابع `handle_xxx_command(client, message, group_guid, user_guid)`
  در ماژول مربوطه بسازید و آن را به لیست `_GROUP_COMMAND_HANDLERS` در
  `core/events.py` اضافه کنید.
- برای افزودن سطح دسترسی یا محدودیت جدید، `core/permissions.py` را
  ویرایش کنید.
- تمام تماس‌های خام با rubpy در `core/client.py` متمرکز است؛ اگر
  نسخه‌ی جدید rubpy نام متدی را تغییر داد، فقط همین فایل نیاز به
  به‌روزرسانی دارد.
